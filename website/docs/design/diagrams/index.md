# Diagrams

<Lead>Two visual cuts through the same platform: the states a payment moves through, and the calls that move it.</Lead>

- **[State Diagrams](./state-diagram.md)** — the lifecycle taken apart one workflow at a time: the states each workflow drives and where it hands off to the next.
- **[Sequence Diagrams](./sequence-diagram.md)** — the end-to-end traces: caller → core API → router → workflow → the activity groups that do the work, for all twelve flows.
