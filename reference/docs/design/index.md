# Design

- [**Payment State Model**](./payment-state-model.md) — the single canonical lifecycle every payment moves through.
- [**Journeys**](./journeys/apis.md) — end-to-end paths from APIs and schedulers into workflows.
- [**Workflows**](./workflows/core.md) — core, composite, scheduled, and event-handler workflows.
- [**Services**](./services.md) — the activity-level building blocks that workflows compose.
- [**Database**](./database.md) — how state is persisted.
- [**Diagrams**](./diagrams/state-diagram.md) — state and sequence diagrams.
